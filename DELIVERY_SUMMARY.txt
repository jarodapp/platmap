================================================================================
                            KEY FEATURES
================================================================================

SCRIPT 1: Download HMDA Data
  • Downloads from CFPB Data Browser API
  • Coverage: 51 entities (50 states + DC) × 7 years (2018-2024)
  • Total: 357 CSV files (~3-5 GB)
  • Features: Rate limiting (1 req/sec), retry logic (3x), validation
  • Resumable: Yes (caches downloads, safe to re-run)
  • Time: 6-10 hours

SCRIPT 2: Process HMDA Data
  • Loads all CSVs for each year
  • Race classification: Hispanic > Asian > Black > White > Multi
  • Aggregation: Tract → County → State → National
  • Computes: Counts, weighted-median income, race percentages
  • Output: Compact JSON files (28 total, 50-100 MB)
  • Time: 5-15 minutes

SCRIPT 3: Download & Build Shapefiles
  • Downloads Census TIGER shapefiles (2010, 2020 epochs)
  • Merges 51 state shapefiles into single national GeoJSON
  • Dissolves counties to create state boundaries
  • Simplifies geometries for web performance
  • Output: 4 GeoJSON files + temporary downloads folder
  • Time: 20-40 minutes

SCRIPT 4: Build PMTiles
  • Uses tippecanoe (open-source tile generator)
  • Creates 4 tile layers (states, counties, tracts_2010, tracts_2020)
  • Zoom levels: z2-z14 (world to individual tract)
  • Density optimization for performance
  • Output: 4 PMTiles files (450-600 MB total)
  • Time: 30-60 minutes

SCRIPT 5: Generate Frontend Data Files
  • Reads processed aggregates
  • Writes per-year statistics (states, counties, tracts)
  • Aggregates national homebuyer counts & income
  • Generates metadata and year index
  • Output: 14+ JSON files (25-30 MB total)
  • Time: 1-2 minutes

================================================================================
                          DATA SPECIFICATIONS
================================================================================

RACE CLASSIFICATION:
  1. Hispanic:  derived_ethnicity == "Hispanic or Latino"
  2. Asian:    derived_race == "Asian" AND NOT hispanic
  3. Black:    derived_race == "Black or African American" AND NOT hispanic
  4. White:    derived_race == "White" AND NOT hispanic
  5. Multi:    derived_race in ["2 or more minority races", "Joint"]

Dominant race is highest percentage, or "Highly Diverse" if max < 40%

GEOGRAPHIC LEVELS:
  • Tract:  11-char GEOID (state + county + tract)
  • County: 5-char FIPS (state + county code)
  • State:  2-char FIPS code

JSON OUTPUT FORMAT (per geography):
  {
    "geoid_or_fips": {
      "r": "race",           # dominant race or "Highly Diverse"
      "tx": 1234,            # transaction count
      "inc": 75000,          # weighted median income
      "pw": 0.65,            # percent white
      "pb": 0.15,            # percent black
      "pa": 0.06,            # percent asian
      "ph": 0.12,            # percent hispanic
      "pm": 0.02,            # percent multi
      "iw": 80000,           # median income: white (optional)
      "ib": 55000,           # median income: black (optional)
      ...                    # ... (optional per-race income)
    }
  }
